## 3. 반드시 모아야 할 시각 자료

가능한 자료에 체크한다.

- 최종 UI 화면
- before / after 화면
- user flow
- IA / sitemap
- wireframe
- prototype 링크 또는 영상
- design system / component library
- handoff spec
- QA 문서
- Notion/Jira 운영 화면
- stakeholder presentation
- research notes
- metric dashboard
- testimonial / recommendation

## 4. Big Tech 기준으로 특히 중요한 증거

각 프로젝트에 아래 증거가 있으면 강하다.

- Ambiguous problem을 어떻게 구조화했는가
- Complex workflow를 어떻게 단순화했는가
- 사용자의 실제 pain point는 무엇이었는가
- business goal과 user goal을 어떻게 연결했는가
- engineering constraint를 어떻게 반영했는가
- PM/engineer/researcher/stakeholder와 어떻게 협업했는가
- 어떤 prototype을 만들고 무엇을 검증했는가
- accessibility 또는 inclusive design 고려가 있었는가
- design system/component 재사용이 있었는가
- 출시 후 metric 또는 qualitative signal이 있었는가

## 5. 프로젝트별 우선 질문

### Butler

- Retention 40% 증가는 어떤 기준으로 측정했는가? 근거 설명 가능.
- landing/home redesign 전후 화면이 있는가?
- 어떤 UX 문제가 retention에 영향을 주고 있었는가?
- 어떤 디자인 결정이 가장 큰 영향을 만들었는가?
- Jira/Notion/hybrid process는 공개 가능한가?

### SM Meta-passport

- 공개 가능한 최종 화면이 있는가? UI 공개 가능.
- membership 구조와 핵심 user flow를 설명할 수 있는가?
- 사용자는 누구였고, 가장 중요한 task는 무엇이었는가?
- 이해관계자는 누구였고, 어떤 제약이 있었는가?
- Erik이 직접 설계한 영역은 어디인가?

### Bero AI

- 공개 가능한 product concept 또는 prototype이 있는가? 방향성과 기능 화면 공개 가능.
- 아직 런칭하지 않았으므로 in-progress case로 표현한다.
- AI가 해결하려는 사용자 문제는 무엇인가?
- research에서 얻은 핵심 인사이트는 무엇인가?
- AI interaction model을 어떻게 설계했는가?
- trust, safety, ethics 관점에서 고려한 것이 있는가?

### Brickmate Design Operations

- 실제 guideline/template/component 예시를 익명화해서 보여줄 수 있는가? 익명화 공개 가능.
- 팀 규모나 프로세스가 어떻게 바뀌었는가?
- onboarding/hiring/training process 중 공개 가능한 것이 있는가?
- design QA/QC 또는 handoff process가 있었는가?
- $1.2M+ delivery와 $10M M&A 관련 수치를 공개해도 되는가?
