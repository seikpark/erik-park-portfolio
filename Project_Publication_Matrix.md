## Confirmed Target

- Market: North America
- Role: Product Designer
- Target level: Senior Product Designer 정도
- Level nuance: 영어 커뮤니케이션이 원어민 수준은 아닌 점을 감안해 Lead/Manager보다 Senior IC 중심으로 포지셔닝
- Differentiator: PM/Product Owner/Design Manager 경험을 product judgment, stakeholder alignment, delivery ownership으로 번역

## Communication Strategy

Senior 레벨을 노리려면 포트폴리오가 다음을 대신 증명해야 한다.

- 복잡한 문제를 구조화할 수 있음
- 사용자의 pain point와 business goal을 연결할 수 있음
- 디자인 의사결정의 이유를 명확히 설명할 수 있음
- PM/engineer/stakeholder와 협업하며 출시까지 고려할 수 있음
- 영어가 아주 화려하지 않아도 짧고 명확한 문장으로 설득할 수 있음

Writing principle:

- Short sentences
- Clear section titles
- Strong visuals first
- One idea per paragraph
- Avoid clever but vague copy
- Use metrics and artifacts instead of overexplaining

## Case Study Implications

### 1. Butler

Status:

- Retention +40% 근거 설명 가능.

Portfolio angle:

> Improving retention through a clearer fintech product experience.

Needed next:

- Retention metric definition
- Measurement period
- Before/after landing and home screens
- Key UX changes
- PM/process artifacts only if they support the design story

### 2. SM Meta-passport

Status:

- UI 공개 가능.

Portfolio angle:

> Designing a global membership experience for entertainment fans.

Needed next:

- Final UI screens
- User flow
- Membership structure
- IA or service map
- Erik's direct contribution
- Stakeholder constraints

### 3. Bero AI

Status:

- 방향성과 기능 화면 공개 가능.
- 아직 런칭 전.

Portfolio angle:

> Designing an AI communication experience from ambiguity to product direction.

Important wording:

- "Current work"
- "In progress"
- "Early product direction"
- "Prototype / feature exploration"
- "Not yet launched"

Avoid:

- Claiming product impact
- Claiming user outcome
- Making the case sound finished if it is still in development

Needed next:

- Problem framing
- Product direction
- Feature screens
- AI interaction model
- Research inputs
- Trust/safety or ethical considerations if relevant
- Next steps

### 4. Brickmate Design Operations

Status:

- 자료 익명화 후 공개 가능.

Portfolio angle:

> Building design systems and workflows for consistent product delivery.

Needed next:

- Anonymized guideline/template examples
- Component or design system examples
- Before/after workflow
- QA/QC or handoff process
- Team enablement evidence
- Public-safe delivery scale metrics
